Numeric values

Calculation result is generally a numeric value, except the result of some converter functions, which have a result by more than one value, so are represented by a reverse function.

Both integers and decimals can be either written or represented in Scientific Notation (i.e. 1.5013093754529656e+68).

Integers

Integers are limited only by System memory, nevertheless SUM stops calculation before it overflows the memory or CPU.

Large integer result is represented in Scientific Notation by default. To disable that option, uncheck "Large integer result in Scientific Notation" in Preferences - "Calculations" group.

Decimals

Decimals, also known as floats or floating point numbers, are limited by 1.7976931348623157e±308. Anyway, decimal value results in natural representation are rounded to 15 digits after floating point.

Unlike integers, too large and too small decimals are always represented in Scientific Notation.

Binary, Octal, Hexadecimal

Special types of binary, octal and hexadecimal values are defined by "0b...", "0o..." and "0x..." prefixes respectively. These special types can be used in any place of ordinary numeric values. The only exception of using special type is only one type allowed for the same expression.

Expressions with a special type value has a result of the same type. If the input consists of a value of a special type, the decimal equivalent will be returned.

See also bin, oct and hex functions, to convert a decimal value to a special type.
